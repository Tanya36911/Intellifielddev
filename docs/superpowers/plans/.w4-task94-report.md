## How AssignPanel handles useParams vs bare MemoryRouter

In real routing (full App), `useParams()` returns `{ id: 's1' }` from the `:id/assign` Route. In isolated test renders (bare MemoryRouter without a matching Route), `useParams()` returns `{}`. AssignPanel falls back to parsing `location.pathname` with a regex to extract the survey id.

## Button Loading State Pattern

The Assign button shows "Loading..." (disabled) while either the nodes query or the survey query is still in flight. It shows "Assign" (enabled) only after both queries resolve and a published version exists. This allows `findByRole('button', { name: /^assign$/i })` to poll until data is loaded, matching the test's intent.

# W4 Task 94 - Code Review Fix Report

Date: 2026-06-24
Commit: 85eb75e

## Fixes Applied

### Fix 1 - Partial-failure retry must not duplicate assignments (Important)
Added `succeededRef = useRef<Set<string>>(new Set())` to `AssignPanel.tsx`. The assign loop now skips any nodeId already in the set, adds each id after a successful POST, and only navigates away once all succeed. A partial failure leaves already-succeeded ids in the set so the next click only POSTs the remainder.

### Fix 2 - Loading guard for the "no published version" banner (Important)
Changed the banner condition from `!nodesLoading` to `!isDataLoading`. The component-level `isDataLoading` flag covers both the nodes query and the survey query, so the banner no longer flashes while the survey fetch is still in flight.

### Fix 3 - hasSelection must reflect actual node ids (Important)
Changed `hasSelection` from `allStores || selectedOthers.size > 0` to `selectedNodeIds.length > 0`. `selectedNodeIds` is computed from the actual nodes response, so when the root node is absent (empty nodes response) the Assign button stays disabled.

### Fix 4 - Partial date/time deadline (Important)
Added a `deadlineState` variable: both-filled returns the UTC ISO string, both-blank returns `null`, exactly-one-filled returns `'partial'`. `handleAssign` now returns early with "Enter both a date and a time for the deadline, or leave both blank." when `deadlineState === 'partial'`. Timezone parsing is unchanged (`new Date(\`${date}T${time}\`).toISOString()`).

### Fix 5 - Test asserts the version id (Important)
Updated `AssignPanel.test.tsx` to include `survey_version_id: 'ver1'` in the `expect.objectContaining(...)` matcher for the POST assertion.

### Fix 6 - Deterministic root node (Minor)
Replaced `Array.prototype.reduce` tie-breaking with a `compareNodes` sort function. Preference order: `parent_id === null` first, then lowest `level_order`, then shortest `path`, then lowest `id` lexicographically.

### Fix 7 - Extract shared pickPublishedVersionId helper (Minor)
Added `export function pickPublishedVersionId(versions: SurveyVersion[]): string | null` to `useSurveys.ts`. Updated `AssignPanel.tsx` to import and use it (removing the private copy). Updated `Builder.tsx` to import and use it (replacing the inline `reduce` in `handleConfirmPublish`). Added three unit tests in `useSurveys.test.ts`: draft+two-published returns highest, no-published returns null, empty array returns null.
