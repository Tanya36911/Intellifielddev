## Deviations from Plan

1. **Line picker rendered outside the collapsible settings panel.** The plan places the line picker inside the settings area. The Task 7 test renders a `perSku: true` question and immediately clicks "Velvet Lip" without opening settings. To satisfy the test without modifying the test, the line picker is now always visible when `q.perSku` is true (rendered after the settings toggle, not inside the toggle panel). The "Required to submit" and "Ask per product" switches remain inside the settings panel. This matches the spirit of the plan (the picker is always accessible when per-product mode is on) and is better UX.

2. **`mapToBackendQuestion` signature.** The plan's test imports `mapToBackendQuestion(q, catalog)` with two args, but the already-committed `useSurveys.ts` accepts `mapToBackendQuestion(q, _catalog?)` with the catalog as an optional param (never re-derives skuIds from it, sends verbatim). No change needed; the call in `QuestionCard` never calls `mapToBackendQuestion` directly.

3. **Prototype file not found.** `hi-fi-intelli/project/apps/admin/screens/formbuilder-parts.jsx` does not exist in the repo. Visual conventions were inferred from the production UI kit components and the existing Catalog screen CSS patterns.
